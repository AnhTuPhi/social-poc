Three Strategies

The hot-key problem: a viral post hits 10k likes/sec. A single DB row + per-row lock means every increment queues behind the lock. Sharded counter spreads writes across N rows; sum on read. Redis-style holds the count in memory and periodically flushes — fastest writes but small data-loss window if Redis dies before flush.
